Designing compelling RPG quests and writing consistent NPC dialogue is one of the most time-consuming and expensive parts of game development. Hard-coded dialog trees limit player agency, but exposing raw AI generation directly to game clients runs the risk of lore conflicts, prompt injection hacks, and game balance exploits.

[1:00 — ARCHITECTURE]
(show assets/architecture_diagram.png)
Let's take a look at the system architecture. When a player prompts the game for a quest or NPC interaction, the request first hits the Security Checkpoint node. This node sanitizes the input, scrubbing PII and checking for cheat codes or prompt injections.
If the query fails the check, the workflow routes to the Security Failure node, which returns a safe denial message.
If cleared, the input routes to the Orchestrator node, which coordinates the generation. The orchestrator has a Model Context Protocol or MCP server connected as a side panel, giving it real-time tools to fetch NPC profiles, campaign quest logs, and item rarities.
If the quest contains a request for a legendary item, a Human Reviewer node pauses the workflow via a RequestInput interrupt, requiring manual confirmation before the quest blueprint is finalized. Once approved, the output flows to the Final Output node and is sent to the client.

[3:00 — SECURITY & MCP]
Under the hood, the security checkpoint implements regex-based PII scrubbing, prompt injection keyword matching, and a content filter for standard game cheat codes, producing JSON audit logs for complete visibility. The Model Context Protocol server uses FastMCP to expose three standalone tools that let the agent pull lore profiles and quest progress dynamically.
